9//! This library provides access to metadata and libraries written by the [Serato
10//! DJ](https://serato.com/dj) software.
11//!
12//! # Metadata
13//!
14//! Serato's formats are pretty complex and store redundant data which might lead to situation
15//! where that data contradicts each other (e.g. if the data was edited by hand).
16//!
17//! ## Parsing
18//!
19//! To make it straightforward to access the metadata, this library provides
20//! the [`TagContainer` struct](`tag::TagContainer`), which provides access to all important
21//! attributes and implements the same conflict resolution strategies that the Serato DJ software
22//! uses.
23//!
24//! ```
25//! use triseratops::tag::{TagContainer, TagFormat};
26//!
27//! fn parse_and_print_cues(markers_data: &[u8], markers2_data: &[u8]) {
28//!     let mut tags = TagContainer::new();
29//!     tags.parse_markers(markers_data, TagFormat::ID3).expect("Failed to parse Markers data!");
30//!     tags.parse_markers2(markers2_data, TagFormat::ID3).expect("Failed to parse Markers2 data!");
31//!
32//!     for cue in tags.cues() {
33//!         println!("{:?}", cue);
34//!     }
35//! }
36//! ```
37//!
38//! If you'd rather parse the tag data representation yourself, you can do that by using the
39//! individual tag structs directly:
40//!
41//! ```
42//! use triseratops::tag::{Markers, format::id3::ID3Tag};
43//!
44//! fn parse_and_print_markers(data: &[u8]) {
45//!     let markers = Markers::parse_id3(data).expect("Failed to parse data!");
46//!     println!("{:?}", markers);
47//! }
48//! ```
49//!
50//! **Note:** This library does *not* provide means to read the metadata from music files directly, so you need to
51//! use other libraries (e.g. [`id3`](https://lib.rs/crates/id3) to do that. You can check the
52//! `examples/` directory for some toy examples.
53//!
54//! ## Serialization
55//!
56//! This library aims to provide support for lossless data roundtripping, i.e. parsing and then
57//! serializing data results in the exact same bytes as the original input (except for FLAC/MP4
58//! data, see last bullet point in the *Caveats* section below).
59//!
60//! ```
61//! use std::io::Write;
62//! use triseratops::tag::{Markers, format::id3::ID3Tag};
63//!
64//! fn write(mut writer: impl Write, markers: &Markers) {
65//!     let bytes_written = markers.write_id3(&mut writer).expect("Failed to serialize data!");
66//!     println!("Wrote {} bytes", bytes_written);
67//! }
68//! ```
69//!
70//! ## Supported File Types
71//!
72//! Support for the following tags has already been implemented:
73//!
74//! | Tag          | ID3     | FLAC    | MP4/M4A | Ogg Vorbis | XML (e.g. AAC) | *Description*
75//! | ------------ | ------- | ------- | ------- | ---------- | -------------- | ----------
76//! | `Analysis`   | **Yes** | **Yes** | **Yes** | **Yes**    | No             | Serato Analysis version
77//! | `Autotags`   | **Yes** | **Yes** | **Yes** | No         | No             | BPM and Gain values
78//! | `BeatGrid`   | **Yes** | **Yes** | **Yes** | No         | No             | Beatgrid Markers
79//! | `Markers_`   | **Yes** | *n/a*   | **Yes** | No         | No             | Hotcues, Saved Loops, etc.
80//! | `Markers2`   | **Yes** | **Yes** | **Yes** | **Yes**    | No             | Hotcues, Saved Loops, etc.
81//! | `Offsets_`   | No      | No      | No      | No         | No             | ?
82//! | `Overview`   | **Yes** | **Yes** | **Yes** | No         | No             | Overview Waveform data
83//! | `RelVol`     | *n/a*   | Partial | Partial | *n/a*      | No             | Relative Volume Adjument data (?)
84//! | `VideoAssoc` | *n/a*   | Partial | Partial | *n/a*      | No             | Video Association data (?)
85//!
86//! ## Caveats
87//!
88//! - Most Ogg tags are currently not supported. Their format is completely different from
89//!   the other tag types and need to be reverse-engineered first.
90//! - The `Serato Offsets_` tag haven't been reverse engineed yet, and no support has been
91//!   implemented.
92//! - The `Serato RelVolAd` and the `Serato VidAssoc` tags haven't been reverse engineed yet, but
93//!   preliminary support has been added. For now, they just return a tag version and a byte vector.
94//! - AAC files (among others) do not store metadata in tags, and use XML files in the
95//!   `_Serato_/Metadata` directory instead. No support has been added yet.
96//! - The cue colors stored in the metadata are *not* the same as displayed in Serato DJ Pro.
97//!   Instead, they uses the color palette from Serato DJ Into. Serato then maps them to a new
98//!   color palette. Support for converting between the two is currently missing.
99//! - The track colors stores in the metadata are those from the color picker, not those shown in
100//!   the library table. Support for converting between the two is currently missing.
101//! - Unfortunately, full lossless roundtrips  aren't possible for FLAC and MP4, because the data
102//!   is wrapped in base64-encoding where the decoded last byte seems to be random junk that change
103//!   when writing tags from Serato DJ even if no actual changes were made (possibly an
104//!   out-of-bounds read or uninitialized data in Serato DJ). For parsing and using this in Serato
105//!   that doesn't make a difference, but the roundtrip tests will ignore those last two bytes.
106//!
107//! # Library
108//!
109//! Parsing the Serato library (e.g. the `database V2` file in the `_Serato_` directory) is also
110//! possible, but since this feature is still under development, the API is *not* stable yet and
111//! might change in the future.
